Automated Cryptocurrency Trading Bot

Python 3 · Binance API · PostgreSQL · Docker · Raspberry Pi Cluster · 2025

I developed an automated trading bot that analyzes cryptocurrency market data and executes trades based on algorithmic strategies. The system collects real-time data from the Binance API, applies technical indicators, and performs automated buy/sell operations using Python scripts and a PostgreSQL database for storing historical data.

  • Implemented strategy modules with configurable risk parameters and integrated them with Binance spot trading.
  • Added a backtesting engine that replays historical data to validate algorithms before live deployment.
  • Containerized services with Docker and orchestrated monitoring dashboards on a Raspberry Pi cluster.
  • Built alerting and health checks to keep the bot fault tolerant when network or exchange issues occur.

This project taught me how to design stable, fault-tolerant systems where data accuracy and timing are critical.

ptf_nn_agent for Windows

Python 3 · Ethernet · TCP/IP · Sockets · Device Validation · 2024

Developed ptf_nn_agent, a Windows-compatible extension for the Packet Test Framework (PTF) to automate L2/L3 network verification. The agent bridges PTF scripts with the Windows networking stack through zero-copy NanoMsg sockets, unlocking cross-platform testing and device validation.

  • Bridged PTF scripts with native Windows adapters, translating Ethernet frames and routing metadata for on-host validation.
  • Implemented NanoMsg-based IPC so PTF traffic generators remotely control Windows adapters.

KeePass-Compatible Password Manager

C++ · Qt5 · Cryptography · Cross-Platform · 2021

I built a lightweight, cross-platform password manager fully compatible with the KeePass (KDBX) format. It supports master-password and key-file authentication, secure entry management, and local encryption using AES-256 with Argon2id key derivation and HMAC-SHA256 integrity protection. Source code available on GitHub.

  • Implemented the KDBX 4 data model, including groups, entries, custom fields, and attachment handling.
  • Integrated Qt5 UI components for cross-platform desktop support with secure clipboard management.

Custom ls Implementation

C · POSIX API · Mac · Linux · System Calls · 2020

Implemented a custom version of the Unix ls utility in C, replicating core functionality such as directory traversal, sorting, and permission parsing using POSIX system calls. The project deepened my understanding of low-level file system APIs, buffers, and process I/O management.

  • Handled option parsing for flags such as -l, -a, -t, -R, -A, and -p to mirror GNU ls behavior.
  • Used opendir, readdir, and stat to gather metadata and format human-readable output.
  • Implemented platform-aware abstractions to ensure compatibility across macOS and Linux environments.

Source code available on GitHub.
